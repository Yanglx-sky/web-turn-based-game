# Web回合制精灵游戏 - 项目文档

## 📋 项目概述

**Web Turn-Based Game** 是一款基于Web的精灵回合制对战游戏，采用前后端分离架构。玩家可以选择御三家精灵，通过PVE关卡挑战、训练模式提升实力，体验AI智能对战、社交聊天等丰富功能。

### 核心特色
- 🎮 **回合制战斗系统**：基于速度先后的策略对战，支持技能释放、精灵切换、药品使用
- 🤖 **AI智能集成**：大模型驱动的怪物嘲讽、战斗策略推荐、战后AI总结
- 🧚 **精灵收集养成**：御三家精灵选择、等级提升、技能学习、装备系统
- 📊 **关卡挑战系统**：多关卡PVE挑战，AI评分评星（0-3星）
- 🏋️ **训练模式**：自定义训练人偶，支持属性配置、主动/被动攻击
- 💬 **社交系统**：世界频道、私聊、好友系统
- 🏆 **排行榜与成就**：多维度排名竞争、成就进度追踪
- 🔒 **安全机制**：JWT认证、战斗防并发、AI调用限流、敏感词过滤

## 🏗️ 技术架构

### 后端技术栈
- **核心框架**：Spring Boot 3.2.0 + Java 17

- **实时通信**：WebSocket（聊天系统）

## 🎯 核心功能模块详解

### 1. 战斗系统 ⚔️

#### 1.1 PVE关卡战斗
**核心逻辑**：
- **速度优先机制**：比较双方速度值决定出手顺序
- **属性克制系统**：火(1) > 草(3) > 水(2) > 火(1)，克制系数1.5倍伤害
- **技能系统**：MP消耗管理、技能类型（物理/特殊/变化）
- **精灵切换**：支持3只精灵出战，死亡自动切换，切换时恢复状态
- **回合制流程**：玩家行动 → 怪物行动 → 回合结算 → 判断胜负
- **战斗日志**：持久化到`battle_log`表，支持AI分析和断线重连查看

**关键接口**（`/battle`）：
| 接口 | 方法 | 说明 |
|------|------|------|
| `/battle` | POST | 开始战斗（传入levelId） |
| `/battle` | GET | 获取当前战斗状态（断线重连） |
| `/battle/action` | POST | 执行战斗动作（attack/skill/switch/monster_turn） |
| `/battle/status` | PUT | 更新战斗状态（offline/online/flee/abandon） |
| `/battle/battle-elves` | GET | 获取战斗中的精灵列表 |
| `/battle/submit` | POST | 提交战斗行动（带回合校验和幂等性） |
| `/battle/reward` | POST | 领取战斗奖励（验证胜利标记） |
| `/battle/logs` | GET | 获取战斗日志 |

**战斗状态枚举**：
- `status = 0`：进行中
- `status = 1`：胜利
- `status = 2`：失败
- `status = 3`：逃跑/放弃

**安全机制**：
- **防并发**：`BattleSecurityInterceptor` 防止同时发起多场战斗
- **回合校验**：`submitAction` 接口校验回合号，防止重放攻击
- **幂等性**：Redis记录已处理的回合动作，防止重复提交
- **超时处理**：定时任务（每5分钟）清理超时战斗（`@Scheduled(cron="0 */5 * * * ?")`）

#### 1.2 训练模式
**功能特点**：
- **自定义训练人偶**：可配置攻击、防御、血量、MP、速度、系别、是否反击
- **满血训练**：每次训练精灵满血满蓝开始
- **与PVE一致的战斗逻辑**：同样的速度判定、技能选择、属性克制
- **训练日志**：独立记录，不持久化到battleRecord（仅PVE持久化）
- **暂停/恢复**：支持离线暂停、重连恢复

**关键接口**（`/train`）：
| 接口 | 方法 | 说明 |
|------|------|------|
| `/train/create` | POST | 创建训练人偶 |
| `/train/start` | POST | 开始训练（支持直接传参或使用已有木偶） |
| `/train/attack` | POST | 普通攻击 |
| `/train/skill` | POST | 使用技能 |
| `/train/monster_turn` | POST | 执行人偶行动 |
| `/train/switch` | POST | 切换精灵 |
| `/train/use_potion` | POST | 使用药品 |
| `/train/settlement` | POST | 训练结算 |
| `/train/flee` | POST | 逃跑 |
| `/train/offline` | PUT | 玩家离线（暂停训练） |
| `/train/records` | GET | 获取训练记录 |
| `/train/battle_elves` | GET | 获取训练中的精灵 |
| `/train/logs` | GET | 获取训练日志 |

#### 1.3 AI怪物嘲讽 🗣️
**实现机制**：
- **动态生成**：基于大模型API实时生成个性化对话
- **状态感知**：
  - 怪物血量<30%：愤怒、恐惧、垂死挣扎
  - 玩家血量<30%：得意、嘲讽、轻蔑
  - 正常状态：常规战斗对话
- **概率触发**：70%概率在攻击/被攻击时触发
- **格式规范**：强制要求"怪物名+动作词+内容"格式，确保前端识别
- **容错机制**：AI失败时自动切换硬编码嘲讽库（30+条预设文本）

**前端处理**：`Battle.vue.extractEnemyDialog()` 方法解析嘲讽并显示

### 2. 精灵系统 🧚

#### 2.1 精灵管理
- **御三家选择**：
  - 火系（焰阳火灵，ID=1）：高攻击、中等速度
  - 水系（惊涛水灵，ID=2）：均衡属性
  - 草系（魔草巫灵，ID=3）：高防御、高血量
- **属性成长**：等级、经验、攻击、防御、速度、HP、MP
- **技能学习**：根据系别自动学习对应技能（`elf_skill`关联表）
- **出战配置**：设置3只出战精灵及顺序（`fight_order`字段）

**关键接口**：
| 接口 | 方法 | 说明 |
|------|------|------|
| `/elves` | GET | 获取所有精灵模板 |
| `/elves/starter` | GET | 获取御三家精灵 |
| `/user-elves` | GET | 获取用户拥有的精灵 |
| `/user-elves/battle` | GET | 获取出战精灵 |
| `/user-elves/config` | PUT | 配置出战顺序 |

#### 2.2 装备系统
- **装备类型**：武器、防具、饰品等
- **属性加成**：提升攻击、防御、血量等
- **装备穿戴/卸下**：动态调整精灵属性
- **商店购买**：消耗金币购买装备

**关键接口**（`/equip`）：
| 接口 | 方法 | 说明 |
|------|------|------|
| `/equip/list` | GET | 获取装备列表 |
| `/equip/wear` | POST | 穿戴装备 |
| `/equip/remove` | POST | 卸下装备 |

#### 2.3 精灵状态管理
**多表同步机制**：
- `user_elf`：永久数据（等级、经验、属性）
- `battle_record_elf`：战斗临时数据（当前HP、MP、状态）

**精灵状态枚举**（`elfState`）：
- `0`：战斗中（当前在场）
- `1`：可上场（未死亡）
- `2`：已死亡（本场战斗不可再用）

**精灵正常状态**（`status`）：
- `1`：正常
- `0`：冻结/异常

### 3. 关卡系统 🗺️

#### 3.1 关卡解锁
- **线性解锁**：通关当前关卡解锁下一关
- **关卡信息**：怪物配置、奖励经验/金币
- **进度持久化**：`user.current_level` 记录已通关最高关卡

#### 3.2 AI评分评星 ⭐
**触发时机**：关卡胜利后自动调用

**评分标准**：
| 分数范围 | 星级 | 回合数 | 描述 |
|---------|------|--------|------|
| 90-100 | 3星 | ≤5回合 | 完美通关 |
| 75-89 | 2星 | 6-10回合 | 顺利通关 |
| 60-74 | 1星 | 11-15回合 | 勉强通关 |
| 0-59 | 0星 | >15回合 | 非常艰难 |

**评分要素**：
- 回合数（主要因素）
- 战斗效率（技能运用、属性克制）
- 精灵存活状况

**数据存储**：`user_level_star` 表记录每关最高星级（只增不减）

#### 3.3 战斗总结 📝
- **AI生成**：基于完整战斗日志生成战后分析
- **内容包括**：
  - 关键转折点分析
  - 胜负原因总结
  - 改进建议
  - 精灵成长状况
- **不持久化**：实时生成，不存储到数据库（节省存储空间）

## 🔐 安全与权限机制

### 认证机制
- **JWT Token**：用户登录后生成Token，有效期24小时
- **请求头传递**：`Authorization: Bearer <token>`
- **拦截器验证**：`AuthLoginInterceptor` 统一拦截验证
- **白名单机制**：登录、注册接口无需认证

### 战斗安全
- **防并发攻击**：`BattleSecurityInterceptor` 防止同时发起多场战斗
- **状态校验**：战斗结束后自动清理战斗状态
- **回合校验**：提交行动时校验回合号，防止重放攻击
- **幂等性保障**：Redis记录已处理动作，防止重复提交
- **数据验证**：关键操作（切换精灵、使用技能）进行服务端校验

### 限流策略
- **AI调用限流**：Redis计数器，每日20次上限（`AILimitInterceptor`）
- **敏感词过滤**：Redis缓存敏感词库，24小时过期
- **战斗超时清理**：定时任务每5分钟清理超时战斗

### 日志记录
- **@Loggable注解**：标记需要记录日志的Controller
- **LogInterceptor**：记录请求方法、路径、参数、耗时
- **全局异常处理**：`GlobalExceptionAspect` 统一捕获异常并记录

## 🔧 开发规范

### 后端规范
1. **Controller层**：仅处理HTTP请求，返回`Result<T>`统一响应格式
2. **Service层**：业务逻辑实现，使用`@Transactional`管理事务
3. **Mapper层**：数据库操作，继承`BaseMapper<T>`
4. **日志规范**：使用`@Slf4j` + SLF4J占位符（`log.info("用户ID: {}", userId)`）
5. **异常处理**：使用全局异常切面`GlobalExceptionAspect`
6. **Swagger文档**：所有接口必须添加`@Tag`、`@Operation`、`@Parameter`注解
7. **JWT密钥**：必须Base64解码后使用（`JwtUtil`中已处理）

### 前端规范
1. **组件命名**：PascalCase（如`BattleReconnectDialog.vue`）
2. **API封装**：统一在`src/api/`目录下管理
3. **状态管理**：使用Pinia，复杂状态持久化到localStorage
4. **样式规范**：使用CSS变量，支持主题切换
5. **路由配置**：需要认证的页面添加`meta: { requiresAuth: true }`

## 🐛 历史Bug修复记录

### 战斗系统修复
1. **战斗重连逻辑修复**
   - **问题**：重连查询条件仅`status=0`，无法重连逃跑/放弃的战斗
   - **修复**：修改为`(status=0 OR status=3)`，同步更新注释
   - **文件**：`BattleRecordMapper.xml`、`BattleRecordMapper.java`

2. **金币奖励补全**
   - **问题**：战斗胜利后未发放金币奖励
   - **修复**：在`normalAttack`和`skillAttack`方法中增加`userService.addGold()`调用
   - **文件**：`BattleServiceImpl.java`

3. **战斗状态清理**
   - **问题**：战斗结束后未清理Redis战斗状态，导致无法发起新战斗
   - **修复**：胜利后调用`battleSecurityInterceptor.updateBattleStatus()`清理状态
   - **文件**：`BattleServiceImpl.java`

4. **普通攻击日志完善**
   - **问题**：普通攻击未记录伤害和HP变化
   - **修复**：添加伤害计算、HP变化、回合日志结构化输出
   - **文件**：`BattleServiceImpl.java`

5. **战斗超时定时任务**
   - **问题**：超时战斗未自动清理
   - **修复**：添加`@Scheduled(cron="0 */5 * * * ?")`注解启用`handleTimeoutBattles()`
   - **文件**：`BattleServiceImpl.java`

### 精灵系统修复
6. **精灵升级MP恢复**
   - **问题**：精灵升级后MP未恢复到最大值
   - **修复**：在升级逻辑中补充`userElf.setCurrentMp(userElf.getMaxMp())`
   - **文件**：`UserElfServiceImpl.java`

7. **精灵状态初始化**
   - **问题**：创建精灵时`status`字段未初始化
   - **修复**：在`createElf`中设置`userElf.setStatus(1)`（正常）
   - **文件**：`UserElfServiceImpl.java`

### 事务与安全修复
8. **商店事务管理**
   - **问题**：购买装备未使用事务，可能导致数据不一致
   - **修复**：为`ShopServiceImpl.purchaseEquip`方法添加`@Transactional`注解
   - **文件**：`ShopServiceImpl.java`

9. **JWT密钥修正**
   - **问题**：JWT密钥未正确解码，导致Token验证失败
   - **修复**：在`JwtUtil`中将`SECRET_KEY` Base64解码为字节数组后传入
   - **文件**：`JwtUtil.java`

10. **权限配置修复**
    - **问题**：部分接口被Spring Security拦截，返回403错误
    - **修复**：调整`SecurityConfig`配置，放行公开接口，确保JWT拦截器正常工作
    - **文件**：`SecurityConfig.java`、`AuthLoginInterceptor.java`
